* HTML 이란?
- 웹브라우저가 알아들을수 있는 문서의 형식(html문서)
- html문서는 태그 <tag>로 구성됨.
- 태그란? html문서를 구성하는 명령어
- <태그명></태그명>
- 태그작성시 소문자 사용

=================

*html문서의 기본형식
<html>
     <head>
        <title>웹문서제목</title>
     </head>
     <body>
        웹브라우저에 출력될 내용 영역
     </body>
</html>

*주석문
코드에 작성하지만, 웹브라우저 상에 출력되지 않는 영역.
html, css문서의 종류에 따라 주석의 종류도 달라짐.
주석문 안에 또다른 주석문 포함되어지면 오류남.
html문서에 css주석 적으면 오류 / css문서안에 html주석 작성시 오류

주석의 형태 2가지
<!--html주석문 -->
/* css주석문 */

*기본태그
<div></div>                : 웹 ui의 영억을 나누는 역할(블록요소)

<em></em> : span보다 작은 개념. 전문용어, 번역문구 (인라인요소/기울임꼴), 웹접근성 때문에 사용

== display 속성 ==
1) block요소 (블록요소)
- 자동 줄바꿈, 세로정렬
- 기본 가로너비값을 전체 100% 인식(부모상속)
- width, height 공간값 적용 가능
<div> <p> <ul> <li> <ol> <dl> <dt> <dd> <h1 ~ h6>

2) inline요소 (인라인요소)
- 가로정렬, 줄바꿈 안 됨
- 기본 가로너비값을 자신의 안쪽에 작성된 글자 or 이미지 양만큼 인식
- width, height 공간값 적용 안 됨
- 주로 내용(컨텐츠) 담당할 때 사용하는 속성
<a> <span> <strong> <i> <em> ...

3) {display:inline-block}
- 기본은 inline 요소
- 인라인 요소이면서 block의 (가로, 세로 공간값 인식) 특징 가짐
- float 대신 대체해서 사용하지는 않음(가로배열할 때 특정영역에서 사용)
- <img> <input> ...

4) {display:none}
- 해당 속성을 가진 태그는 html 구조상에서 삭제되는 형식
- 공간도 없어지고, ui 화면출력도 되지 않는 상태
- 주로 ui user interaction 효과에 사용함


5) {display:block} 은 2가지 형식으로 사용됨
1. 인라인 요소인 태그를 block요소화 시켜 공간값을 인식시키는 개념
2. {display:none}으로 사라진 태그를 {display:block} 으로 보이게 할 때 사용

<a></a> : 링크 역할 (인라인요소)

* href 필수 속성 : 링크 이동경로 지정
<a href="이동경로"></a>

1) 절대경로 : http:// ~~
                  C:\user\doument\index.html
2) 상대경로 : 현재 작성하는 html 문서를 기준으로 경로 지정.
같은 곳에 있다면 : 파일명.확장자
폴더 안에 들어가 있다면 : 폴더명/파일명.확장자
내(현 html)가 나와서 찾아야 한다면 : ../

/ root루트 최상위폴더
./ 현재 위치 기준
../ 상위폴더

target = "_blank" : 새 탭에서 열기
target = "_self" : 기본값. 현재 창에서 열기

_parent
_top

<img> : 이미지의 경로를 연결시키는 태그 (인라인 요소)

*src 필수속성 : 이미지 경로 지정
<img src="이미지 경로">

*alt 속성 : 웹접근성 위해서 이미지 설명글 제공
               (정사출력 경우 글자 보이지 않음. 안 보일 때 화면에 글자 출력)
<img src="이미지경로" alt="이미지설명글">

* 목록태그 3가지 : 모두 블록요소 세로정렬하는 것이 특징.

1) <ol> : 순서의 의미
2) <ul> : 순서에 상관없이
3) <dl> : 정의형 목록태그

: ol, ul 모두 <li>태그를 포함
: <dl>
   <dt>대표단어</dt>
   <dd>대표단어 설명글</dd>
</dl>
